Settings > Software options: Placement process (advanced)

The following settings are for test purposes only!

Watchdog options

In order to provide a meaningful bug report if the machine comes to a standstill during an endurance run, the report is created automatically as soon as the X or Y axis of the selected gantry does not move during the specified interval.

Select the gantry for which you want to monitor movement of the X and Y axes.

Monitoring is enabled.

Monitoring can be started by selecting the axes when an endurance run or the "Simulate placement of components" function is started and as soon as all the axes are moving.

Monitoring is terminated if the machine is stopped or a bug report is automatically generated.

Timeout [s]

Time interval in seconds: If a selected axis does not move after this time has elapsed, a bug report is created automatically.

Hint: In the case of "Simulate placement of components", the time interval set here must be sufficiently large to accommodate the longest pauses for an axis defined in the placement program. Otherwise, a bug report is created and monitoring is terminated.

Default: 300 s.

Simulate placement of components

The placement operations are performed for test purposes, but no components are placed (dry run).

Dry run mode

Dry run

The placement process is executed for test purposes without placing components.

Feeder position recognition is also carried out and communication with the required feeders is tested. However, no components are provided by the feeders.

Simulate setup

Dry run without feeders:

Feeder position recognition is not carried out and communication with the feeders is not tested.

This option can be enabled if not all feeders or no feeders are physically present or if no component tables are present.

Use available feeders (Only displayed for the "SIPLACE service" activity level.)

A dry run can be performed irrespective of whether individual feeders or component tables are present.

Feeder position recognition is carried out for the available feeders and communication is tested. Missing feeders are ignored.

Board handling

Standard board transport

The board moves through the machine in the same way as when components are placed.

No board transport

The board remains in the processing section and is not transported on.

Cyclic board transport

During placement simulation, the board is moved from the input section into the placement section and then into the output section. When each cycle is complete, the board is transported back into the input section and the process starts again.

This option is used to test clamping, positioning and transport of the boards during the placement test.

Automatically set full

If a simulated feeder runs out of components, i.e. if the components that were theoretically present are set to 0, the number of components for a full feeder is automatically assumed.

Endurance run with components

Overnight endurance run: A board is moved in and a corresponding placement job is specified from SIPLACE Pro. After all the components have been placed, the placement head picks them up again and returns them to the appropriate waffle pack tray. The machine then starts to place the components again.

Return placed components after finishing board

The components are returned to the waffle pack tray.

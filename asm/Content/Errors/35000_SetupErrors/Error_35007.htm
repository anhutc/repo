35007 Device and location recognition failed

The component pocket could not be recognized.

Check to see whether the correct feeder has been set up on the corresponding track and if necessary, teach the component pocket again.

If the pickup position is already set correctly, but feeder position recognition can still not be performed:

Skip feeder position recognition for this feeder (detail view of X feeder).

Fiducial on the component table is dirty or damaged.

Clean the fiducial and if necessary replace the component table.

The table was not found: Table fiducial was not recognized.

Start the view Service > Teach machine positions > Inspect positions with gantry, choose the relevant fiducial (see sender) and click Move to position.

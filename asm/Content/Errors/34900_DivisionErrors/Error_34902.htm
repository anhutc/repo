34902 Pickup location deactivated because of frequent pickup errors

There was regularly no component on the nozzle after pickup (no component was picked up). On the C&P20 and CPP heads,the presence of a component is checked using the component sensor, and on the Twin Head and C&P12 head, it is checked using the vacuum values.

At the specified pickup position, the proportion of pickup operations that return an error is greater than the specified threshold value.

Use the error list under Current messages > Statistics to check whether the pickup errors are particularly frequent for a specific segment.

 

If this is the case:

Check the nozzle and the segment:

C&P20 head, CPP head, Twin Head, C&P12 head

If the errors are reasonably evenly distributed across all segments,

check the specified feeder:

Make sure that the pickup position for the feeder is set correctly and that the component is picked up correctly.

  • In the case of components with dimensions greater than a 0402 package, switch to the Correct pickup position view and change the settings if necessary.
  • In the case of smaller components, switch to the Correct pickup position view and make sure that the component pocket has been described correctly.

    Recommendation: Use the default settings for measuring the component pocket.
  • Check that the component is being picked up with the correct nozzle.
  • In SIPLACE Pro, check the setting for the height of the component sensor, the acceleration values and the travel profile settings.

 

Then press the Start button .

The error counter is reset.

Flux inspection

 

In order to ensure that all connections (balls or columns) have been coated with flux after BGA components have been dipped, the Vision system inspects the connections after dipping.

This is done by capturing and saving an image after the position has been determined and after the component has been dipped in flux.

These two images are then compared: The brightness values are compared, pixel by pixel, at the points at which connections have been defined.

If the average brightness difference for a connection is below a given threshold (Minimum difference from the reference image), the connection is deemed to be insufficiently coated.

By default, a component is deemed to be NOK if at least 1 connection has been identified as insufficiently coated (parameter: Maximum number of uncoated connections).

If a higher parameter value is specified, you can also define the minimum required distance between 2 uncoated connections (parameter: Minimum gap between uncoated connections). This allows you to ensure that a region is not incorrectly identified as OK (e.g. as a result of a creased component or if the flux is not distributed evenly).

Flux inspection can only be successful if a suitable flux or solder paste is used which causes a sufficient change in appearance between dipped and undipped components. To ensure that flux is detected at all, it is made "visible" by the addition of dark or light pigments.

As a minimum, the following cases should be tested in the component shape manipulator:

A component is not dipped, but flux inspection is activated.

Result: Flux inspection fails, in other words all connections should be flagged as insufficiently coated.

A component is dipped when flux inspection is activated.

Result: Flux inspection succeeds, in other words all connections should be flagged as sufficiently coated.

If these results are not achieved, the parameters must be adjusted.

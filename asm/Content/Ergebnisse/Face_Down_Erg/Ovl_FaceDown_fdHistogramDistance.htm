Recognition step: Detection of flipped components - histogram and classification when selecting the contrast method

 

Blue lines

Brightness distribution in the region of interest.

Pale blue frame

Standard deviation of the brightness distribution.

Yellow frame

Fixed, specified threshold for the standard deviation.

Green horizontal bar

If the standard deviation is in this region, the component is detected as having been picked up correctly.

Red horizontal bar

This indicates the region in which the component is detected as having been picked up with the underside facing up.

Example 1:

 

All components whose standard deviation is greater than the specified threshold are marked as having been picked up with the underside facing up: The light blue border is larger than the yellow border.

 

The component was picked up with the underside facing up.

 

 

Example 2: 

 

The standard deviation is smaller than the threshold value. The light blue border is inside the yellow border.

 

The component was picked up correctly.

 

Representation of the results of the recognition of components which have been picked up upside down if the contrast method is selected and the top and underside have been taught:

Red vertical lines

Reference histogram determined during teaching in the event that the component is identified as having been picked up with the underside facing up.

Green vertical lines

Reference histogram determined during teaching in the event that the component is identified as having been picked up correctly.

Blue vertical lines

Histogram for the current component.

Blue vertical bar

Shows the degree of correlation with one of the reference histograms, i.e. the brightness distribution in the region of interest determined during teaching.

Red vertical area

Component has been picked up with the underside facing up.

Green vertical area

Component has been picked up correctly.

 

Example 1:

 

The histogram for the current component matches the histogram for components which have been picked up with the underside facing up (the blue bar is in the red area).

 

The component has been picked up with the underside facing up and is therefore rejected.

 

 

Example 2: 

 

The histogram for the current component matches the histogram for components which have been picked up correctly (the blue vertical bar is in the green area).

 

The component has been picked up correctly and can be placed.

Example 3: 

The blue bar in the green area is higher than the blue bar in the red area. This means that the correlation in the green area (picked up correctly) is greater.

 

The component has been picked up correctly and can be placed.

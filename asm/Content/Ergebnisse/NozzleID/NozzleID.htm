Autodetection of the nozzle type

Depending on the selection you make as to whether nozzle type detection is performed for one or more garages, the board camera is positioned over a single garage, over all the garages of a specific magazine in sequence or over all the garages of the nozzle changer and nozzle detection is started.

The system checks whether there is a nozzle in the garage and whether this nozzle has a nozzle code. If a nozzle with a nozzle code is detected, the nozzle type can be determined and entered under "Current" in the table in the Nozzle changer - Magazine functions view.

 

The following results are possible for the evaluation:

The garage is occupied, and the nozzle code is recognized.

The nozzle type detected is entered under "Current". The corresponding garage is highlighted with a blue dot in the representation of the magazine.

The garage is occupied, but there is no nozzle code.

The previous occupancy remains displayed.

The garage is empty.

An empty garage is shown in the detail view and "-" is shown in the table.

The garage is occupied, and the nozzle code is faulty.

The previous occupancy remains displayed and an error message is issued. A negative quality value is issued.

The garage is occupied and the nozzle code is detected, but this cannot be assigned to a nozzle type.

The previous occupancy remains displayed and an error message is issued.

 

Nozzle code:

The nozzle code is a special pattern of barcodes arranged at angles to each other. The barcodes are implemented using encoding bumps:

Model

Camera image

The annular area in which the encoding bumps are located is divided into 17 areas of the same size.

Each of these areas corresponds to one bit of the code:

1

There is an encoding bump in this area

0

No encoding bump

One of these areas is reserved for the start bit, which specifies the beginning of the code. This ensures that the nozzles can be uniquely identified, even if they are twisted in the garage.

The start bit is implemented using a special encoding bump which is wider than all the rest.

The remaining areas represent the data bits which are used to determine the nozzle ID. When error detection and correction are used, this means that 256 different codes can be detected. Because (0000 0000 0000 0000) is not used, the actual number of different nozzle IDs is 255.

The assignment of the individual nozzle IDs to the nozzle type is stored in a table. This allows the nozzle type to be determined.

Inkspots

 

Inkspots are used for identifying boards or panels. This marking enables the machine to decide during placement whether a board should have components placed on it or if it should be omitted. There is a choice of stickers, ink marks, holes, punched holes or circuit structures as ink spots. It is, however also possible to create ink spots by drawing over structures with a marker pen.

Before an ink spot can be used as a decision criterion, a model of the ink spot must be created, describing its geometrical form.

If it is not possible to describe the geometrical form, the ink spot is taught, i.e. the structure and orientation of the ink spot are captured by the board camera and the data derived from this is stored. It is necessary to describe both ink spots identifying panels which are to have components placed on them ("go" case) and ink spots identifying panels which are not to have components placed on them ("no-go" case). In the event that an ink spot cannot be clearly identified during placement (system is uncertain), the operator decides how to deal with panels which have not been definitively flagged.

The standard method is to use geometrically described ink spots.

Problems when evaluating ink spots.

The structures must be of a good quality:

The machine does not come to a standstill if a structure is not found. Instead, the panel or board is not assembled. If it happens that "good" panels or boards are not assembled because the structure was not recognized, you can adjust the settings for the algorithm parameters. Then test whether a "bad" panel or board is also actually omitted.

The quality shown does not correspond to the measured quality of the structure, but is calculated from the discrepancy between the measured quality and the minimum quality
The greater the discrepancy between the two values, the more reliable the quality is for ink spots, i.e. the quality shown is high.

During placement, the board is loaded and ink spot recognition is carried out. Then the stored data is compared with the actual ink spot and a decision is made regarding whether components should be placed on the panel or not.

The following work has to be carried out:

  1. Start inkspot processing.
  2. Ink spot description: Specifying the size and shape of the ink spot or
    Teaching ink spots using board camera images.
  3. Testing ink spots

 

Three different methods are currently used for classifying taught ink spots. An appropriate method has to be selected depending on the nature of the ink spot.

Template method

Histogram method

Mean grayscale value

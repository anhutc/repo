Orientation

Components which are rotated by 180째, 90째, 60째 or 45째 in the feeder cannot be measured and therefore also not be placed, as their actual position deviates too much from the expected position. Such components are rejected. When placing expensive components, this should, however, be avoided wherever possible. For this reason, a function has been provided which allows components to be placed which are lying at a different angle.

 

If the system detects that a component is rotated, a warning to this effect is issued. This may be an indication of incorrect setup data. It is necessary to check the settings.

 

If orientation detection is enabled (2, 4, 6 or 8) and if the system detects that a component is rotated, it is possible to correct the rotation of the component before placement to allow it to be placed correctly.

In the Twin Head, however, this additional check can lead to significant delays.

On the C&P20 head, it is possible that placement accuracy will be negatively impacted if the component is not presented in the expected orientation.

 

Recommendation:

Only activate orientation detection if:

  • you expect that components may be presented with an incorrect orientation in the feeder (components in trays),
  • it is more important to place all components than to achieve the shortest possible placement times,
  • the arrangement of the leads allows the orientation to be determined.

 

If orientation detection is enabled, make sure that inspection mode has not been disabled.

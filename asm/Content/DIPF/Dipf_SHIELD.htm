Component shape: Shield

These are metal sheets used as shields (some with drilled holes). The body forms vary considerably (polygons).

 

The following parameters describe a component shape of the type "Shield":

Parameter

Description

Global properties

Name

Name of the component shape.

User text

Comment specified in the programming system.

1

Body length = overall length

2

Body width  = overall width

Body height = overall height

 

Number of connections

The total number of all described features that have to be recognized by SIPLACE Vision.

Separate ICOS group description

Yes

The component shape has separate group descriptions for ICOS that cannot be changed in the component shape manipulator.

Recommended setting, because SIPLACE Vision is able to use improved measurement procedures to detect features that cannot be evaluated in ICOS.

No

No separate group descriptions are available.

Special measurement options

Orientation

 

Checking the orientation with the stationary camera degrades placement performance.

1

Default:
The system does not check whether a component is rotated by 180°, 90°, 60° or 45° in the feeder.
If a component has been picked up at an incorrect orientation, and its position cannot be measured due to asymmetrically arranged connections, it is rejected.

Caution: The system cannot detect rotation on components with symmetrically arranged connections.

If components which are rotated in the feeder are to be placed, you must specify how many different positions the component can have in the feeder.

2

Rotation through 180° can be detected (2 different positions).

4, 6, 8

4, 6 or 8 different positions for the component can be distinguished.

Suppress checking of threshold values

Yes

Checking of the minimum and maximum values specific to a given sensor system type is not carried out.

If checking of the threshold values is suppressed, this may negatively impact on the recognition rate and the accuracy. This may cause unexpected errors.

No

The check is carried out.

 

Features

Definition of corners

Definition of the corners in the X and Y view: 2 lines are defined that lead to the corners.

Corner (n):

1

Point 1

2

Point 2

3

Point 3

4

Point 4

The X and Y coordinates of each point must be specified.

Note: Points (2 and 3) must not be located on the corner to be detected.

Definition of the corners in the Length, radius, angle view

1

Length 1: Length of line 1 (line connecting points 1 and 2), minimum length

2

Radius 1: Distance between line 1 and the corner

3

Radius 2: Distance between line 2 and the corner

4

Length 2: Length of line 2 (line connecting points 3 and 4), minimum length

5

Corner angle (angle between the x-axis and the first line, anti-clockwise)

6

Corner: Coordinates in X and Y

7

Opening angle (angle between line 2 and line 1, anti-clockwise)

Description of rectangular structures

X/Y

Coordinates of the center of the rectangle.

Angle

Angle of rotation of the rectangle with reference to the component.

Length / Width

External dimensions of the rectangular structure.

Maximum tolerance for length and width: 20 % of the corresponding nominal value.

Feature is light

Yes

Description of a rectangular area that appears bright compared with the body of the component or the complete component (contour measurement).

No

 

Description of a rectangular punched hole or of a rectangular dark area.

Definition of drilled holes, punched holes

Diameter

Diameter of drilled hole or external contour.

Feature is light:

Yes

This setting is used to describe a shield with a rounded external contour.

No

Description of a drilled hole in a shield

 

  • The positions of the defined corners must match the data sheet exactly. If this is not the case, it can lead to inaccuracies during placement.
  • The lengths of line 1 and line 2 should be similar.

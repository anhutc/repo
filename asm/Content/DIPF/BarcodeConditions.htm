Barcode conditions

In order to prevent barcodes from being confused, up to three conditions can also be set when teaching a barcode. Only when all the defined conditions have been met is the barcode recognized, otherwise the system continues to search for a valid barcode in the search area.

Activating/deactivating barcode conditions:

  • As soon as one of the following three possible conditions for recognizing the barcode has been changed, the barcode conditions are taken to be set (on).

  • If you want to deactivate the conditions again, all changes must be reset:
    This is done by pressing Reset for the text lengths or by deleting the entry made under Expression.

Barcode conditions

Minimum code text length

Minimum number of bytes the barcode must contain.

Maximum code text length

Maximum number of bytes the barcode may contain.
The default value (2000) corresponds to the maximum number of bytes that can be processed.

Expression

Here, you can define a string that the barcode must match in order for it to be valid.

Press the corresponding button and enter the string.

Use text expression with "*" and "?"

Default: Text strings can be entered using wildcards.

Permitted wildcards:

*: Any string

?: Any single character

Use regular expression

A string can be entered that describes sets of strings using special syntax rules.

Characters of the normal alphabet are used in combination with metacharacters (e.g. [ ] ( ) { } | ? + - * ^ $ \ ).

Component shape: Bare die

The following parameters describe a component shape of the type "Bare die":

Rectangular body, no leads (only length, width and height):

Parameter

Description

Global properties

Name

Name of the component shape.

User text

Comment specified in the programming system.

1

Length of the component (X) = body length X

2

Width of the component (Y) = body width (Y)

3

Height of the component (Z) = body height (Z)

 

Separate ICOS group description

Recommendations

Yes

The component shape has separate group descriptions for ICOS that cannot be changed in the component shape manipulator.

No

No separate group descriptions are available.

Special measurement options

Detection of flipped components

Yes

Various methods are available to identify whether a component has been picked up correctly or with the bottom side facing up (flipped).

No

This inspection is not carried out.

Suppress checking of threshold values

 

Yes

Checking of the minimum and maximum values specific to a given sensor system type is not carried out.

If checking of the threshold values is suppressed, this may negatively impact on the recognition rate and the accuracy. This may cause unexpected errors.

No

The check is carried out.

Chipping inspection

Yes

The edges of the component are inspected for signs of splintering or chipping.

No

This inspection is not carried out.

 

Special case: Components picked up from a SIPLACE BulkFeeder X.

The following settings only apply for detection using the camera at the BulkFeeder X.

Flipped components

Detection of flipped components

Yes

The brightness values of the top and bottom of a component are to be determined in order to identify whether a component has been picked up correctly or with the bottom side facing up (flipped).

No

This inspection is not carried out.

Brightness threshold

 

Default: 127.

Threshold for distinguishing light areas from dark areas: If the measured value is above the specified brightness threshold, the rectangle is recognized as "light".

Flipped if brightness is above threshold

Yes

For components where the top is lighter than the bottom side.

No

This is the default for components where the top is darker than the bottom side.

The component is recognized as having been picked up correctly if the measured brightness (bottom side) is above the threshold.

X / Y extent of region of interest

Length and width of the ROI in [mm].

Brightness method for detection of flipped components

To allow the detection of flipped components, the mean brightness of a rectangle in the center of a component is determined. The result is given in brightness levels (range: 0 - 255).

 

A threshold is specified known as the brightness threshold.

  • If the measured value is above the specified brightness threshold, the rectangle is recognized as "light".
  • If the measured value is below the specified brightness threshold, the rectangle is recognized as "dark".
  • Default setting: 127

 

Tips for using this method:

  • If the component is white on one side and black, green or blue on the other side, teaching will generally only need to be carried out once for the light side in most applications.

  • However, if the component is white on one side and red, orange or yellow on the other, it is necessary to pick up and teach at least one correct and one flipped component.

  • If the component shape is used on several stations, the teach operation should also be repeated on the other stations, without deleting the existing data. An ideal brightness threshold will then be calculated from all the teaching data.

Detection of flipped components: Results if gullwing connections are evaluated

 

Teach current component:

Specify whether the component currently shown is in the correct orientation or is flipped:

OK

 

The component is in the correct orientation and can be placed.

Flipped

 

The component is flipped and cannot be placed.

 

Trained components

Two components can be shown here: The most recently taught component in the correct orientation and the most recently taught flipped component.

For each component, the brightness value determined for the region of interest is specified. (This value is averaged across the regions of interest of the different connections).

Example: Brightness threshold = 139, Flipped if brightness is above threshold: Yes

OK

The component is in the correct orientation. The brightness value determined in the regions of interest is 99.

Flipped

The component is flipped. The brightness value determined in the regions of interest is 179. The region is thus brighter than the specified brightness threshold.

To the right of the images, up to 10 values are shown. These are the most recent 10 brightness values of the taught components.

Region of interest

Yellow frame

Regions of interest: The area for which the brightness is determined in each gullwing lead.

Histogram and classification (example)

Blue lines

Brightness distribution in the region of interest.

Cyan vertical line

Average brightness in the region of interest of the current component.

Yellow vertical line

Fixed brightness threshold.

Green horizontal bar

If the mean brightness is in this region, the component is detected as having been picked up correctly.

Red horizontal bar

This indicates the region in which the component is detected as having been picked up with the underside facing up.

Result

Representation of the position and orientation of the component as determined, together with the information whether the component is in the correct orientation or flipped.

Green frame

Representation of component.

Green coordinate system

The component is identified as being in the correct orientation and can be placed.

Red coordinate system

The component is identified as being flipped and cannot be placed.

Parameter

Value

range

Description

Brightness threshold

0..255

Default: 127.

Threshold for distinguishing light areas from dark areas: If the measured value is above the specified brightness threshold, the rectangle is recognized as "light".

Flipped if brightness is above threshold

Yes/No

Yes

A component is classified as flipped if the region of interest is lighter than the specified threshold.

No

The component is recognized as having been picked up correctly if the region of interest is darker than the specified threshold.

Back

Switches back to the Define region of interest view.

Accept (Only shown if parameters have been changed.)

Parameters that have been changed are saved and the wizard is closed.

OK (Only shown if no parameters have been changed.)

The current settings are retained and the wizard is closed.

Cancel

The wizard is closed and any changes made are rejected.

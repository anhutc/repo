Illumination optimization

Optimized illumination can be set automatically for measurements with the component camera. This provides the highest possible contrast between the connections and the background.

The aim is to make the measurement more robust and thus reduce the number of rejected components.

 

Requirements:

  • The components to be measured must be of the type "Nonstandard" or "Connector".
  • The "OSC package" license must be available on the programming system.

Before starting optimization:

  • Completely describe all the features: Specify the feature type and, where necessary, 'Feature is light' (yes/no).
    The features in the model should match those shown in the camera image.

  • The position of the component must be known.
    If this is not the case, you should move the model so that it covers the representation in the camera image.

After optimization has been started:

  • During optimization, the measurement with the best setting to date is shown.

After optimization has finished:

  • When illumination optimization is complete, the measurement using the best illumination setting is shown.

  • You can then decide whether or not to accept this setting

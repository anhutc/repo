Pin-in-paste height check

As soon as the pin-in-paste height check is enabled for a component shape in the programming system, the following process is initiated when a board is loaded:

  • Height check for the placed component is automatically selected for all placement positions assigned to this component shape.

  • The first board is automatically flagged for inspection. When it has been assembled, it stops at the position entered under Settings > Machine options Inspection. The status of each placement position with pin-in-paste height checking is "undefined Z reference position" and the position must be confirmed.

  • Pin-in-paste confirmation: For each of these placement positions, you must specify whether or not placement is OK, i.e. the board is removed and a check is carried out for each placement position to establish whether the centering pins have fully passed through the board.

    If you specify the status OK, the Z height measured for this placement position (the stored value for the lowest point reached by the Z axis when the component was placed) is taken as the Z reference height and the pin-in-paste height check is enabled for all further boards of this type.

  • The Z reference heights determined in this way are stored in a file along with additional board data (e.g. board name, transport mode, job type, ...). These values can then be re-used if the machine is switched off during production or if the same board is produced again at a later time.

    But as soon as any value in the board data changes, the Z reference heights of the individual placement positions must be determined again.

  • Pin-In-paste height check:
    If the measured difference (2) between the Z height and the Z reference height is greater than the threshold specified in the programming system (pin-in-paste, Z threshold), the component is flagged as being incorrectly placed.

Recognition parameters: Component recognition for components with leads/balls/columns/THT pins (search tree)

Component recognition for components with leads/balls/columns/THT pins is carried out in 2 steps:

  1. Position recognition
    Up to 20 features, which are as far apart as possible, are selected automatically. The position of the component is determined using these features.
  2. Component inspection (prerequisite: exact component position)
    All features are checked for presence, positional tolerance and where appropriate for their properties.
    Whether or not this inspection is carried out can be specified in the component description in the programming system.

 

The following parameters can be set:

Parameter

Default

Description

General settings

 

Minimum quality on position measurement

0.8

The calculated quality value must be greater than the value specified here before the component is flagged as recognized.

Recommendation: 0.6-0.9

Binning

Depends on camera type and component shape

The larger the value that is set here, the faster an image will be loaded and transferred, and the lower the resolution will be.

1

Each pixel is evaluated. Binning is deactivated.

2

2x2 pixels are merged to form one pixel.

4

4x4 pixels are merged to form one pixel.

Optimization for small-pitch connections

Depends on the component size

The filter for detecting individual balls or columns can be set in such a way that even connections that are very close together can still be identified as separate connections.

The preset value is chosen depending on the specified distance between the connections.

On

Connections which are extremely close together will also be detected.

Advantage: Extremely accurate detection.

Off

The connections must be a minimum distance apart in order to be identified as separate connections.

Advantage: Faster detection.

Lead-specific/ball-specific settings

Minimum contrast of feature

40 (leads)

20 (balls)

The contrast between the background and the feature must exceed this value to allow pixels to be assigned to the feature.

Subsampling

1

Increment for overlaying the filter.
The higher the subsampling level, the fewer pixels are used for marking in the filtered image. This leads to a shorter evaluation time.

With large features, it is recommended that a larger subsampling value is used in order to reduce the evaluation time.

Minimum number of pixels in filtered image

5 (leads)

2 (balls)

In the filtered image, those points are displayed light where the difference in grayscale values is greater than the minimum contrast of the feature. Neighboring light points are merged to form regions. If the number of pixels in a region is greater than the defined value, the region is marked as a feature.

Filter size

5 (leads)

6 (balls)

7 (columns)

Size of the feature model which is overlaid over the feature during filtering.

Too large: The outer filter points are located on neighboring structures.

Too small: Not all outer filter points are outside the feature (if the feature and the filter cover each other exactly).

Line width on inspection

7 (leads)

 

When inspecting connections:

A contour is drawn along a line perpendicular to the connection group. The value corresponds to the width of this line.
The line width is calculated from the contact length and the length tolerance of the leads. These values result in a minimum line width below which values cannot fall whatever setting is made.

Note:
If you enter a value less than the calculated minimum line width, this entry is ignored and the minimum line width is used for evaluation.

Settings for balls only

Filter type

20

Filter type 20. 22
For balls which are represented as light features.

Preferred: Type 20 for balls with flat illumination, type 22 for balls without a dark core.

Filter type 32:

For balls which are represented as dark features: Dark, circular objects on a light background.

Detection threshold for balls

 

Ratio:
Number of point pairs used to recognize the ball to the total number of point pairs of a filter.

If the specified value is exceeded, a ball is marked at this position.

Distance between a pair of points

 

The distance between the inner and outer points of a point pair.

Enlargement:

When the connection diameters vary considerably or when the transition appears very fuzzy.

22

Filter type 22

Filter type 32:
For columns which are represented as dark features: Dark, circular objects on a light background.

66 %

Ratio:
Number of point pairs used to recognize the column to the total number of point pairs of a filter.

Settings for the Pattern feature

Minimum contrast of feature

10

 

The contrast between the background and the feature must exceed this value to allow pixels to be assigned to the feature.

Subsampling for coarse search

2

Increment for overlaying the filter.
The higher the subsampling level, the fewer pixels are used for marking in the filtered image. This leads to a shorter evaluation time.

With large features, it is recommended that a larger subsampling value is used in order to reduce the evaluation time.

Detection threshold for polygons

75 %

The quality calculated for a filter must be above this value for a polygon to be recognized.

Quality: Ratio of the point pairs of the filter that are above the "Minimum contrast of feature" threshold to the total number of point pairs of the filter.

Distance between a pair of points

5

The distance between the inner and outer points of a point pair.

Settings for shield components:

Description of shields using corners:

1. Coarse step: The positions of the corners are determined using a polygon filter.
2. Fine step: The positions of the corners are detected using contours.

Edge quality

 

The gradient threshold as of which gradient maxima are used as position candidates.

The higher this value, the higher a local maximum must be before it is used for position recognition.

Increase the value if, for instance, "double edges" are caused by oscillations.

Edge detection method

 

Filter type 248 (default), 249. 250, 251 (binary image)

Special treatment: Wide leads (e.g. with D-Pack).

Detection threshold for polygons

 

The quality calculated for a filter must be above this value for a polygon to be recognized.

Quality: Ratio of the point pairs of the filter that are above the "Minimum contrast of feature" threshold to the total number of point pairs of the filter.

Distance between point pairs for polygon feature

 

The distance between pairs of points.

Increasing the distance reduces the number of point pairs. This leads to higher evaluation speed, but a loss of accuracy and robustness.

Distance between a pair of points

5

The distance between the inner and outer points of a point pair.

Filter type

90 / 91

Filter type 90

Settings for round THT pins and rectangular THT pins

Filter region

2

If this value is greater than 1, corresponding regions around the filter point will be evaluated in addition to the filter point itself.

Advantage: The higher this value, the more tolerant detection will be in respect of fluctuations in shape, and the tips of pins with an appearance that differs considerably from the image that has been learned will also be detected.

Disadvantage: Slightly longer evaluation times.

If too large a value is chosen, it is also possible that artifacts will be identified as the tips of pins.
